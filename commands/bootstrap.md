# /bootstrap — Scaffold New Project

Create a new project folder with all infrastructure from templates.

## Input
$ARGUMENTS — project name (e.g., "clinic-portal")

## Execution

1. Create project folder: `~/projects/{project-name}/`
2. Copy and fill templates:
   - SPEC.md (already generated by /generate-spec)
   - CLAUDE.md from templates/CLAUDE.template.md → fill with project info + stack
   - docker-compose.yml from templates/ → configure for chosen stack
   - pyproject.toml from templates/ → add dependencies for chosen stack
   - .gitignore from templates/
   - .env.example with required env vars
3. Create folder structure per SPEC.md architecture section
4. Git init + initial commit
5. Create GitHub repo (with auth guard):
   - Check: `gh auth status 2>/dev/null`
   - If gh CLI is authenticated: `gh repo create {project-name} --private --source=. --push`
   - If gh CLI is NOT authenticated: skip GitHub and log: "GitHub repo not created — run `gh auth login` to enable GitHub integration"
6. Push initial commit (only if remote was created successfully)

## Output

```
Project scaffolded:
  Location: ~/projects/{project-name}/
  Stack: [chosen stack]
  GitHub: https://github.com/{user}/{project-name}
  Files: SPEC.md, CLAUDE.md, docker-compose.yml, pyproject.toml
  Next: /specify SPEC.md
```
